# Review Protocol

## Daily (10-15 min)
- What did I learn that deserves a durable note instead of staying in chat?
- What confused me twice today?
- Which one thing should become an active-output drill tonight?
- Does the current plan still match what I am actually doing?

## Weekly (30-60 min)
- Run `scripts/kb-lint`.
- Review the last 7-14 log entries in `kb/meta/log.md`.
- Consolidate duplicate ideas or weakly linked notes.
- Turn at least one repeated answer into a durable wiki note.
- Create one "teach it back" note in `kb/human/active-output/`.
- Run `scripts/kb-smoke-test` after workflow or tooling changes.

## Monthly
- Which domains are growing in raw material but not in understanding?
- Which project mistakes are repeating?
- Which notes are collect-only and never used?
- Which beliefs should be stress-tested against better sources?
